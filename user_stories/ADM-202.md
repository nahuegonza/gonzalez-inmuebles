# ADM-202 | Gestión de galería multimedia (Imágenes)

**Tipo:** Historia de Usuario
**Estado:** 🟡 PARCIAL
**Descripción:** Como administrador, quiero subir, ordenar y eliminar fotografías de una propiedad para armar la galería visual que verán los usuarios.

**Criterios de Aceptación:**
* **Escenario 1: Carga masiva de imágenes** 🟡
    * **Dado que** el administrador se encuentra en la pestaña "Multimedia" de una propiedad.
    * **Cuando** arrastra y suelta múltiples archivos de imagen (o usa el botón de carga).
    * **Entonces** el sistema debe subir las imágenes de forma asíncrona, mostrando una barra de progreso por archivo.
    * **Y** debe validar formato (JPG, PNG) y peso máximo (5MB).
    * > **Parcialmente implementado:** Existe integración con el widget de Cloudinary (`cloudName: 'dzyiwuftf'`) en ambos paneles. Se pueden subir múltiples imágenes y se muestran previews. **No hay barra de progreso** por archivo (el widget de Cloudinary lo maneja internamente pero no se muestra custom). **No hay validación explícita de formato/peso** en el código propio (depende de la configuración del upload preset en Cloudinary).
* **Escenario 2: Ordenamiento de la galería** ❌
    * **Dado que** la propiedad ya tiene imágenes cargadas.
    * **Entonces** el administrador puede arrastrar y soltar las miniaturas para cambiar el orden.
    * **Y** la primera imagen será la imagen de portada.
    * > **Falta:** No hay implementación de drag & drop para reordenar imágenes. Las imágenes se agregan en el orden de carga y no pueden reorganizarse.

**Bugs / Pendientes:**
- ❌ No hay drag & drop para reordenar imágenes.
- ❌ No hay barra de progreso de carga individual por archivo.
- ❌ No hay validación de formato/peso en el código del proyecto.
- ⚠️ El `uploadPreset: 'ml_default'` es el preset por defecto de Cloudinary — debería configurarse un preset propio con restricciones.
