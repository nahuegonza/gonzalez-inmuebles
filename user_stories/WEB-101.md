# WEB-101 | Formulario de solicitud de venta de propiedad

**Tipo:** Historia de Usuario
**Estado:** 🟡 PARCIAL
**Descripción:** Como usuario propietario, quiero acceder a un formulario desde la página principal para dejar mis datos y que un agente inmobiliario se contacte conmigo para vender mi propiedad.

**Criterios de Aceptación:**
* **Escenario 1: Acceso al formulario de venta** ✅
    * **Dado que** el usuario se encuentra en la Home del sitio.
    * **Cuando** hace clic en el botón "Vender" del menú de acceso rápido.
    * **Entonces** el sistema lo redirige a la vista "Vendé tu propiedad".
    * > **Implementado:** Existe sección `#tasacion` en la home y página `tasacion.html` dedicada. El link de nav "Tasación" redirige correctamente.
* **Escenario 2: Solicitud de permisos de ubicación** ❌
    * **Dado que** el usuario ingresa a la vista del formulario de venta.
    * **Cuando** la página carga.
    * **Entonces** el navegador debe solicitar permisos mediante un prompt nativo para utilizar la ubicación actual del usuario.
    * > **Falta:** No existe ninguna llamada a `navigator.geolocation` en ningún archivo. No se solicita la ubicación al cargar.
* **Escenario 3: Visualización de campos del formulario** ✅
    * **Dado que** el usuario está visualizando el formulario "Vendé tu propiedad".
    * **Entonces** debe visualizar los siguientes campos obligatorios/opcionales:
        * Nombre y apellido (Input text)
        * Email (Input email)
        * Teléfono (Input number/tel con indicación de código de área)
        * Localidad/provincia (Input text)
        * Tipo de propiedad (Dropdown menu)
        * Comentarios (Textarea)
    * > **Implementado:** Todos los campos están presentes en `tasacion.html`. El formulario simula el envío con un `setTimeout` (no hay backend real).

**Bugs / Pendientes:**
- ❌ No se solicita permiso de geolocalización al cargar la página.
- ⚠️ El envío del formulario es simulado (no hay integración con backend/email).
