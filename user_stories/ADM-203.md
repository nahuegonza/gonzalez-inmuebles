# ADM-203 | Bandeja de entrada de consultas de propiedades

**Tipo:** Historia de Usuario
**Estado:** ❌ FALTA
**Descripción:** Como administrador o agente, quiero visualizar un listado con todas las consultas recibidas desde los formularios de las propiedades para darles seguimiento.

**Criterios de Aceptación:**
* **Escenario 1: Visualización de listado de leads** ❌
    * **Dado que** el administrador ingresa a la sección "Consultas".
    * **Entonces** debe ver una tabla con: Fecha, Nombre, Teléfono, Email, Propiedad de interés y Estado de contacto.
    * > **Falta:** Los formularios de contacto (`contacto.html`, `property-detail.html`) solo simulan el envío con un `setTimeout` o `alert()`. No hay persistencia de datos ni backend. La sección "Consultas" en el sidebar del admin (`index.html`) existe como link pero no lleva a ninguna vista implementada.
* **Escenario 2: Detalle del mensaje** ❌
    * **Dado que** el administrador hace clic sobre una consulta específica.
    * **Entonces** se despliega una vista o modal con el mensaje completo y enlaces rápidos.
    * > **Falta:** No existe ninguna implementación de este escenario.

**Bugs / Pendientes:**
- ❌ No hay backend ni persistencia para capturar consultas del formulario.
- ❌ No hay vista de listado de consultas en el panel admin.
- ❌ No hay sistema de estados de contacto (Nuevo / Contactado / Descartado).
- ❌ Los formularios de contacto son completamente decorativos (no envían datos a ningún lado).
