# WEB-103 | Filtro de búsqueda por Ubicación con autocompletado

**Tipo:** Historia de Usuario
**Estado:** ❌ FALTA
**Descripción:** Como usuario buscador, quiero utilizar un buscador por texto con autocompletado en el modal de filtros para encontrar rápidamente la zona de mi interés.

**Criterios de Aceptación:**
* **Escenario 1: Búsqueda predictiva de ubicación** ❌
    * **Dado que** el usuario abrió el modal de "Filtros" e ingresa al menos 3 caracteres en el campo "Ubicación" (ej. "agrono").
    * **Entonces** el sistema debe desplegar una lista de sugerencias coincidentes.
    * > **Falta:** En `properties.html` no existe campo de ubicación en los filtros. El autocompletado via Nominatim solo está implementado en `panel.html` (panel admin) pero no en el frontend de búsqueda.
* **Escenario 2: Selección y conteo dinámico de resultados** ❌
    * **Dado que** el sistema mostró las sugerencias de ubicación.
    * **Cuando** el usuario selecciona una opción.
    * **Entonces** la selección debe mostrarse como un 'tag' o 'chip' con la opción de eliminarla (icono "X").
    * **Y** el CTA debe actualizar el texto con la cantidad de resultados coincidentes.
    * > **Falta:** No existe implementación de tags/chips ni conteo dinámico en ninguna vista.

**Bugs / Pendientes:**
- ❌ No hay campo "Ubicación" en el sidebar de filtros de `properties.html`.
- ❌ No hay autocompletado con Nominatim en el frontend de búsqueda.
- ❌ No hay sistema de tags/chips para mostrar filtros activos.
- ❌ No hay conteo dinámico de resultados en ningún CTA.
